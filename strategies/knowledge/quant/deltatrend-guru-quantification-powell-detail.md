# DeltaTrend's Guru-Quantification Method, and the Full Detail of His Powell Build

Deep-read of "I Coded Powell Trades' Strategy | Pt. 1" (32 min) and "I Coded This Daytrading Guru's Strategy" (Justin Worlin). The Powell video was already summarized in one paragraph in `powell/powell-course-catalog.md` (§Third-party material); this note captures the methodology and the rule-level detail that summary skipped.

## Key findings

- **His quantification method** (CLAIM/process, both videos): watch the guru's own video in real time, transcribe every stated rule into a running pseudo-code note, *flag every assumption out loud* where the source is ambiguous, build in Pine Script as testable modules ("Break it into modules if you're building some sort of stateful strategy and test each one one by one" [powell, 16:35–16:39]), then backtest over thousands of trades and run the prop-firm MC. This is a good-faith adversarial replication protocol — the useful opposite of cherry-picked guru examples.
- **Powell's fib anchoring is inconsistent on camera** (EVIDENCE): within one video Powell anchors the fib to the leg bottom, iterates only the top up on new pivot highs, then later re-anchors the bottom to a *different* pivot low "with no explanation... this is totally hindsight bias, right? Like you're in replay mode here" [powell, 10:50–12:21]. Already noted in the course catalog; the detail here is that the ambiguity forced DeltaTrend to add his own dormancy rule (fib dies when price taps discount or the HTF-PD-array state changes) and to cap unbounded extension [17:16–18:27].
- **The raw ruleset over-trades** (EVIDENCE): a first-pass faithful implementation produced strings of losing "valid" shorts — "These are valid entries based on what Powell has described. Ironically, I'm sure Powell has some rule where he only takes a specified number of trades a day at maximum. So, we'll have to build that" [powell, 25:18–25:31]. He also had to ban same-bar entries off a just-formed PD array and add a stop buffer past the trigger's far edge. Un-stated trade-frequency governors are where discretionary systems hide.
- **Regime filter results** (EVIDENCE, but the transcript is garbled in places — treat exact figures cautiously): raw 5-year backtest in-sample net ≈ $16,685 with max drawdown ≈ −$25,750; a regime filter found by a Python optimization script raised in-sample WR by ~10 pts and cut drawdown; out-of-sample the filter cut max drawdown from ≈ −$16,000 to ≈ −$2,000 and added ≈ +$4,000 net PnL [powell, 28:24–28:40]. **The transcript never states what the filter actually is** — only that it was regime-based and optimizer-selected. The 22.4% figure is the *post-filter* TopStep pass probability, not the filter's own stat.
- **Powell-build prop MC verdict** (EVIDENCE): on a TopStep 150K model — 22.42% pass probability, ~25% payout probability once passed, +$206 net EV per account after fees, ~$1,100 mean payout, ~10 days from pass to payout, and ~55% probability of *timing out* because the low-WR/high-RR shape wins too infrequently [powell, 29:45–30:33]. His verdict: "Is this a strategy that technically makes money over time on prop firm accounts? Yes... Is it something that we'd want to trade in real life with live capital? Definitely not" [30:33–30:46]. A second "Powell-inspired" variant of his own showed higher pass/payout probability and net EV (no numbers given in transcript).
- **The Worlin benchmark** (EVIDENCE): a faithful HTF-liquidity-sweep / manipulation-leg / inverted-FVG strategy, 2018–2025, 3,500 trades, no commission or slippage (deliberately generous): +107% total return vs +311% for buy-and-hold Nasdaq; Sharpe 0.165; Sortino 0.33 — "this strategy made money but so did the market and a lot more" [worlin, 8:48–9:31]. Technically profitable ≠ investable; the benchmark and risk-adjusted lens kill it.
- **Powell direct quote worth keeping** (CLAIM, Powell's own words inside the video): "5-minute entry trigger is better than a 1-minute, by the way" [powell, 19:57] — note this is the *opposite* granularity of Dual46's frozen 1-minute RB trigger; see application §3.

## Details / mechanics

**The exact rules DeltaTrend coded for Powell's "60k in February" setup:**

1. *HTF bias state machine:* mark 1-day rejection blocks; emit state +1 when price taps (not invalidates) a bullish daily RB, −1 for bearish, 0 when invalidated — multiple RBs of the same polarity must be trackable concurrently (his first build only kept the latest and missed setups [14:56–15:59]).
2. *Freeze-leg fib:* once state ≠ 0, find the 5-minute pivot low (for longs) inside the RB range that starts the reactive leg; anchor the fib there; extend the top to each new 5m pivot high; the fib goes dormant when price taps the 0.5 discount or the HTF state changes; unbounded extension capped by assumption (range multiple / ATR multiple — Powell never addresses it [18:34–19:11]).
3. *Discount tolerance:* Powell allows "edging" the 0.5 level without strictly crossing it, so a few-ticks tolerance parameter is required [8:50–9:11].
4. *Entry trigger:* tap of a 5-minute PD array (FVG / order block / rejection block) formed after the discount tap; limit at the trigger range start; **no entry on the bar the PD array forms** (spam filter); stop just past the trigger's far edge plus a tunable buffer.
5. *Target:* nearest 5-minute pivot high/low **within a viable distance band** from entry (his assumption — "we don't want to choose a really, really far away 5-minute high as a target... we want to scan just within some logical range" [22:40–23:03]), else TP at the band's near edge; minimum-R filter tunable.
6. *Governors he had to invent:* max trades/day, spam-entry mute, stop buffer — none stated by Powell.

**Worlin pipeline** (for contrast): 15m timeframe for resting liquidity (FVGs, swing points), lower timeframe for the manipulation leg and its FVGs; signal = LTF inversion of a manipulation-leg FVG; "nearby" liquidity quantified as a user-set % of 14-day ATR; TP at HTF liquidity, stop past opposite liquidity + 1%-of-ATR buffer, else stop set to force ≥1:1 RR; a sweep's entries expire after N bars.

## APPLICATION TO THE VAULT

1. **The Vault's Dual46 mechanization already solved what DeltaTrend had to invent.** Every governor he bolted on (trade cap, no same-bar entry, stop buffer, target discipline) has a frozen Dual46 equivalent (2–3 trade cap, 1m RB wick-start limit, stop-beyond-extreme + buffer, fixed 1:5/100 pt). Independent confirmation that these governors are load-bearing, not bureaucracy.
2. **Powell RB mechanics without the 10:00 KO context ≈ thin edge.** His faithful build (daily-RB bias, any-session entries) got 22.4% pass / $206 EV / 55% timeout. Dual46 is *not* that system — it adds the 10:00 key-open leave, session restriction, and a high-WR OTE entry. The result doesn't threaten Dual46; it quantifies how little the RB pattern alone carries, which supports the Vault's event-study finding that context (KO leave) does the work. Do not reopen the freeze over any of this.
3. **The 5m-vs-1m trigger quote is a post-May research question, not a change.** Powell now says 5m triggers beat 1m (RR-weighted); Dual46 froze 1m. Log as a backlog event-study variant (B3/B4 territory, one candidate at a time per hard stops) — test only via the standard Stage-0 → path-MC pipeline, never by param retune of killed candidates.
4. **Adopt the "assumption log" convention.** His practice of speaking every quantification assumption aloud maps directly onto the Vault's charter rule of separating claim from evidence. When `analyze-event-study.ts` gets a new candidate spec from discretionary source material, the spec note should carry an explicit ASSUMPTIONS section (tolerances, dormancy rules, distance bands) — these are exactly the free parameters that later become overfitting surface.
5. **Benchmark discipline from the Worlin video:** always report buy-and-hold-the-underlying and risk-adjusted ratios beside raw PnL. The Vault's scorecard is `E[$/wk]`-first, which already dominates raw PnL, but a buy-hold column costs nothing and kills a whole class of "technically profitable" illusions.
6. **His regime-filter result (drawdown −$16k → −$2k OOS) is the strongest third-party datapoint yet** that a single regime gate can transform a marginal intraday system — consistent with `quant/intraday-regime-detection-session-selection.md` and directly relevant to queue topic #32 (vol-regime dependence). Caveat: optimizer-selected filter, one OOS window, undisclosed rule — a lesson about *where* to look, not a number to import.
