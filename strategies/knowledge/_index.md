# Knowledge brain — map of content

**Start here.** Obsidian vault layer for doctrine the agents and you read.
Executable truth lives in `vault-app/lib/` + `strategy-dev/` — notes inform, code decides.

**Product spine (what this brain is for):**
idea → measurable event → Stage-0 EV±CI → path MC `E[$/wk]` → ops → Dual46 / sleeve.

## Is this brain helpful?

**Yes — as curated doctrine, not as a second database.** ~69 notes (~670 KB) already
separate CLAIM vs EVIDENCE and end in `APPLICATION TO THE VAULT`. The app does **not**
parse these files; promotion = distill into `lib/` or `strategy-dev`. Without the brain,
agents re-research the same topics and Dual46 freezes forget *why*. Without a MOC, the
brain becomes a flat dump — that was the real problem, not “too many notes.”
