## 3. Top-down bias (daily → execution)

### Process (keep it this simple)
1. **Daily** — break down highs/lows, PDAs, closes  
2. **4-hour** — same read, finer structure / RBs  
3. **1-hour** — mark **key opens**; refine bias  
4. **15-minute** — what structure is respected vs disrespected  
5. **LTF** — precision entry (RB, fib, triggers, data H/L)

Daily/4H give the **generic directional read**. Hourly + 15m give the **precision bias** you actually hang entries on.

### Previous day high / low (mechanical)
Core swing logic (same family as data H/L):
- **Sweep a side, close back within** → opposing side is typically the **draw**  
  - Sweep high → **lows** should be the draw  
  - Sweep low → **highs** should be the draw  
- Continuation read: **close above** a high → often expect **that high / next high** again; mirror for lows  
- Always blend with **surrounding structure** (not blind every print)

### Discretion at the open (critical)
Ask visually:
1. Which **daily** high/low are we **closest** to?  
2. Which **4H** / **1H** high/low?  
3. What are we most likely to **manipulate into** first?  
4. Where is price **expanding** / where is the **draw on liquidity**?

Example: open closer to the high after rejecting a daily gap with a big wick → it may **not** make sense to run all the way back for previous day low / gap first before the high.

### Signature HTF patterns used for bias
| Pattern | Read |
|---------|------|
| **Daily BPR** | Daily FVG overlapping daily FVG → balanced price range |
| **Daily / 4H rejection block** | Tap level → validating close → later **retest RB → deliver** (signature everywhere) |
| **Equal highs / equal lows** | Engineered liquidity / high-priority draws (speaker especially loves equal lows) |
| **Market maker buy model** (aka V-recovery) | After the model is clear, **do not** keep pressing lower as primary bias; deep retraces can happen but ideally hold nearer the constructive area / EQ–OTE |
| **Daily SMT vs ES** | **Critical** for confirming which equal H/L / extreme is protected |

### Two-scenario honesty
After a daily bearish RB reject that **didn’t tap CE**, both can be valid until price chooses:
- Go **straight for daily lows**, or  
- **Sweep daily high into CE**, then deliver lower  

Range days can **sweep both sides** — don’t force a single-story distribution when tape is ranging.

### Seasonal / context reminder
- Without context, key opens get run  
- ~**April → mid-July** soups needed less context; later tape needs proper HTF story  
- Stacked magnets (NDOG/NWOG) can steal an 18:00 touch — stay level-headed  
- News: check Forex Factory before fading (§5)  
- **Unfilled NWOG/NDOG** with no volume inside = strong free bias toward the fill (§4)

### Loss psychology (bias-adjacent, mandatory mindset)
1. Enter **expecting you can lose** (kills fear-of-entry paralysis).  
2. After a loss: if you instantly have **2–3 excuses** why it failed → it was probably **not** an A setup.  
3. If you **cannot** explain the loss in real time → it was likely a **good setup**; variance / GG.  
4. Hindsight (e.g. “market wanted to range, not distribute”) is fine for review — don’t invent fake real-time invalidations to cope.

Example of a “good loss”: 1m RB + accumulation → manipulate to range EQ + 1m SMT + aiming at barely-manipulated **10 AM** key open — confluence said yes; loss still happens.

## 4. New week / new day opening gaps (NWOG / NDOG)

### Chart setup
- Indicator: **ICT NWOG by LuxAlgo** (only indicator called out)  
- Keep ~**last 4–5** NWOGs visible; strip other indicator noise  

### Core behavior
Price **refers back** to these gaps constantly.  
If a gap is **completely empty** (zero trading / no volume inside): treat fill as **~90–95%** likely — this is framed as a **free bias**. Conflicting opposing bias should usually yield to that unfinished gap.

### Best use (highest quality)
**Jump / rally (or dump) away from the gap without revisiting it** → unfinished gap becomes **free bias into the next session/day**.

Weaker / mid examples: choppy retrace taps, bouncing “randomly” inside/around the gap — still watch, but not the clean free-bias print.

### How to trade them
- Treat similar to a **key open**: prefer waiting for a **candle close above/below** before committing  
- Full fill then expansion is a common delivery path (example ~60pt rally after fill)  
- **Combine** with OBs, FVGs, key opens, fibs — don’t soup isolation if better confluence exists  
- If you don’t want to enter directly on the gap: use the **fib / OB / RB** it creates  
- Many students should prefer **5m entries** over sticky **1m** watching (less panic); set-and-manage is fine  

### Overtrading warning
Many levels are taught on purpose — that can cause overtrading. **Don’t.** Stick to rules, good RR, and the risk cadence in §13.

### Pattern vignette
Rally → rebalance → return to **fill the opening gap** → time spent in/around gaps → later breakout. Getting positioned with the gap fill + confluence is the point.

## 5. AMD model (Accumulation → Manipulation → Distribution)

Framed as one of the **simplest and most effective** standalone models. Posted conceptually with CISD / order-block theory.

### Sequence
1. **Accumulation** — range / consolidation (build liquidity)  
2. **Manipulation** — sweep the side **against** your bias (e.g. bearish → want a push **above** range highs into a PDA)  
3. **Distribution** — deliver in bias direction toward the draw (e.g. range lows / sell-side)

### How he actually takes it
- Bias comes from the **left** (HTF + range story). Example: extremely bearish; **don’t** long a fat rebalance “just because” — that old habit gets run.  
- If the left side is obvious, he may **limit the PDA without waiting** for a fresh CISD/FVG after the tap — “confirmation is to the left of the chart.”  
- Common PDA: **order block** sitting just outside the range (example: 1m OB above range). Prefer **CE** of the OB when refining (5m example).  
- Stop beyond a logical OB fib fraction (example: above **~25%** of the 5m OB ≈ **~10 pts**).  
- TP toward range extreme / first opposing liquidity (example ~**50 pts** to range lows).

### Grading caveat (important)
Same AMD short **inside a daily discount** = **not A+** (shorting discount). Can still take if HTF bias is clearly bearish — expect it to be a **B** grade even when it works.

### Multi-TF
Same model on **1m** and **5m** (Power Hour example was the 5m version of the same AMD + OB/CE idea).

## 7. Key-open “soup” model (standalone focus)

Key opens are a **model by themselves** (same idea as focusing only on wick CEs). Process:

### Step A — Identify the live key open
18:00, midnight, or 10:00 — whichever is relevant to the current narrative.

### Step B — Wait for acceptance (strong close)
Look for a **strong candle close through** the level (above for bullish acceptance, below for bearish).  
This is **not** required to be a formal market structure shift — the close through the open is the tell.

Supporting signs after the close:
- Engineered liquidity / “beautiful” swing to soup  
- FVG present (especially **FVG low** — highly favored)  
- 1m FVG inversions on the bearish side  
- Optional MSS / displacement (nice, not mandatory)

### Step C — Enter the soup (not always on the line)
- Prefer soup of the key open **when context + PD arrays agree**  
- Often **do not enter exactly on the line** — enter the **closest PD array** under/over the open (1m FVG, etc.)  
- Example path: strong close above → pullback into FVG/PDR just below the open → long

### Step D — Invalid / lower-quality soups
- Level with **no context** while a clear opposing draw sits below/above  
- Soup **against** a draw that already completed (wrong bias)  
- Blind line touch with no strong close / no PD array  
- Stacked higher-priority gaps that are likely to absorb the move before your open is delivered

## 8. Data high / low (news bias + post-sweep levels)

**Purpose:** (1) **Free draw / bias** on news, (2) **mechanical opposing-side trade**, (3) after swept, reuse data extremes like **key opens** / wick-CE POIs.

### How the range forms
1. News drops (classic: **8:30**; also **10:00** news days).  
2. The reaction candle(s) print a **data high** and **data low**.  
3. Prefer **aggressive / prominent swings** — real swing, ideally a **big wick** (picture-perfect often clearest on **15s–30s**). Soft little swings are weaker. Equal lows/highs at a data extreme = even more obvious.  
4. High and low are **established when that defining reaction candle closes**.

Data H/L are treated as **stronger / special** vs normal session highs-lows (speaker likens the idea to a POC-style magnet without claiming the academic definition). Prefer **not to trade against** them. Watch **SMT vs ES** — if SMT protects a data extreme, that side may never get swept.

### Core rule A — opposing draw (mechanical)
**First side taken out → target the opposing data pool.**

Example: data high swept first → draw = **data low** (and vice versa).  
On clean news days this draw is “basically free.” **Messy CPI days** can be cancer — first entry after the sweep can fail hard; don’t force.

Do **not** treat a sweep of data low as a primary short while **data high is still untapped** (small scalp only at most — he wouldn’t). Wait for the unfinished opposing side unless you have an exceptional reason.

### Core rule B — after swept, use them like key opens
Once a data extreme is swept / accepted:
- Repeated **tap-and-go / soup** of that white data line is common  
- A **strong close through** it → trade it similarly to a **key open**  
- Can align with **10 AM**, **1m RB**, etc. (example: 10 AM sitting on data low + tiny 1m RB)

### 8:30 open on news days
- **8:30 key open** is mainly for **decent news days**: **CPI, PPI, NFP** (best days to use 8:30)  
- Example path: wait for data high (after data low swept) → later tap **8:30** as the entry POI  

### Entry toolkit after the first side is swept
Use any of the four mechanisms / LTF triggers — PA is often very fast:
| Tool | Example |
|------|---------|
| **30s / 15s CISD** | First 30s CISD after sweep; ~**10 pt** stop → opposing data extreme ≈ **1:4** |
| **IFVG / RB / other** | Allowed once the opposing draw is clear |
| **1m / 5m confirm** | Prefer stronger TF confirm on violent CPI/NFP; if 5m RB is huge → refine with 1m |

Original 1m sequence still valid:
1. Establish data H/L  
2. Sweep one side  
3. Wait for **flip confirmation** (not the sweep alone)  
4. Mark RB (include volume balance); limit or market if already ~1 pt away  
5. Target opposing data extreme  

### Data wick CE (links to wick theory)
If the data candle leaves a **decent wick** (lecture said roughly mid-TF wick; use judgment on 5m-class wicks):
- Can enter the **50% / CE** of that wick  
- **Only if engineered liquidity is present** — that is the **absolute most important** criterion for wick-CE entries. Without it, CE taps are low quality.

### Fundamentals filter (every news drop)
1. Check **Forex Factory**  
2. *Greater than forecast = good for currency* → typically **bad for NQ/ES**  
3. **Mixed prints** → more algo/liquidity-pull → data H/L model fits well  
4. **Severe one-sided beat** (esp. CPI/NFP) → often **trends the rest of the day** — fading early is often bad  

### Grading
- **A+:** Prominent data swing (+ equal H/L nice); first side swept; confirm; clear opposing target; not a cancer tape day; FF not screaming trend-against-you  
- **B:** Soft swing / choppy news but rules followed  
- **Skip:** Trading against unfinished opposing data extreme; SMT protecting target; forcing first soup on atrocious CPI PA; wick CE with **no** engineered liquidity

## 9. Rejection blocks (formal definition)

### What it is
A **rejection block** = the **start of the rejection wick** (effectively that candle’s **open** / wick origin).

Formation:
1. Price **sweeps** built liquidity (highs or lows)  
2. Prints a **rejection wick** (manipulation)  
3. Later **retraces to test** the start of that wick  

Also fine to call it a rejection wick. Works on **every timeframe** from **15 seconds → 4 hours** (premarket, 1H, etc.).

### Entry levels on the RB
| Entry | When to use |
|-------|-------------|
| **Beginning of the wick** (wick start / candle open) | Want higher chance of fill |
| **CE / 50% of the wick** | Risk at the start is too large; tighter invalidation |
| **Wick Theory variant** | Focus **only on wick CE** + **engineered liquidity** (e.g. a low CE) as the classifier for a good trade |

You cannot know in advance whether price will react to an **order block** or the **rejection block** — choose based on **what you are willing to risk**. RB usually = **better (more premium/discount) entry**.

### Best-practice sequence (preferred)
1. RB forms at / into a **POI / area of interest**  
2. Wait for price to leave, then **retest the rejection block**  
3. Enter (wick start or CE)  
4. Stop **beyond the extreme of the wick** that created the RB  
5. Optional confluence: **CISD** (change in state of delivery), **fib golden pocket 0.62–0.79** with a PD array in that pocket  

### Why RBs are used (edge rationale)
- Deliver **very discounted / very premium** entries → better R:R (even when the wick itself is not huge)  
- Psychologically **scary** (large opposing candle into the level) — that is a feature  
- Framing: *if a trade looks easy, you’re about to get hurt*; *the best trades often feel the scariest*  
- Fades the crowd that souped liquidity, got a reaction, then **panics out near BE** when price revisits the wick — RB traders take the continuation after that scare  

### When the RB is too large
If putting the stop at the full wick extreme is a **“hateful” / oversized stop**, do **not** force it — refine (CE entry, lower TF RB, smaller size) or pass. Huge multi-hour wicks are RBs in theory but often unusable at full invalidation.

### 10 AM hybrid (from Transcript 01)
Still core when the session model is **10:00 + RB**:

| Role | Timeframe |
|------|-----------|
| Manipulation filter | **4-hour** wick quality |
| Primary setup | **5-minute** |
| Refinement | **1-minute** |

Session timing:
1. Default wait for **10:00** (framework, not absolute).  
2. Avoid **9:30–10:00** by default.  
3. If **8:30 news**, look then; otherwise wait unless obvious.  
4. Pre-10 only with **strong conviction**.  
5. Mondays / holiday / late-Dec: lower expectancy.

4H wick grades:
| Grade | Look | Action |
|-------|------|--------|
| **Good** | Clear wick (~**15 pts**) | High confidence |
| **Grey** | Soft (~**8 pts**) | Allowed, not preferred |
| **Bad** | Speck (~**2 pts**) | Not enough manip |

Extra 10 AM notes:
- RB at/near 10 AM open → tap zone  
- Or: 10 AM candle **closes below its open** → that **open** usable as OB  
- Prefer liquidity sweep into the open  
- Avoid entries **past fib 0.79** when possible (too premium vs golden pocket 0.62–0.79)

## 10. Entry standards (combined)

### Four entry mechanisms (official list)
For a defined **POI / level** (example: NWOG CE/50 with sell-side balances + engineered liquidity below):

| # | Mechanism | When / how | Stop notes |
|---|-----------|------------|------------|
| **1** | **Straight at the level** | Basically **only valid with engineered liquidity** at/against the level (above or below as appropriate). Limit the level itself. | **5–10 pts** typical, level-dependent |
| **2** | **Rejection block** (**favorite**) | Wick that **rejects** the level + directional confirmation; enter the RB (wick start / “very low of the wick” — **doesn’t have to be 50%**). See §7. | Beyond the RB wick / structure |
| **3** | **CISD** (change in state of delivery) | Enter on the confirmation close; stop beyond the swing made after the level tap (e.g. above the high after a bearish reject). Often lines up with an LTF RB. | Example raw ~3.5 pts → he now prefers **≥ ~5 pts** so a slight sweep of that high can occur |
| **4** | **IFVG** (inverse fair value gap) | After a close that **inverts** the FVG. Entry often at **CE/50% of the IFVG**, or the IFVG overlapping an **HTF OB + OTE** (Models 4). Prefer IFVG as LTF trigger inside HTF PDA. | Beyond IFVG invalidation / structure |

**Stop philosophy update:** used to run **2–3 pt** snipers; now finds **5 pts minimum** is enough and still prints “ridiculous” R (example CISD → first opposing low ≈ **1:8**).

High-confidence when mechanisms **stack** (e.g. 3m RB = also 1m CISD on the same rejection).

### Limits vs entry triggers
Both are valid. Speaker uses **both**; they win in different scenarios.
- **Limits:** fine if RR works and you are profitable with them — including mechanism **#1** when engineered liquidity is present  
- **Entry triggers (#2–4):** confirmation that “saves your ass” (near open, oversized HTF RB, need displacement proof)  
- Choosing precision vs fill odds is **personal** — account type, psychology, max R vs bread-and-butter **1:5–1:6** fills  
- Many students should prefer **5m** over sticky **1m** watching (less panic)

### When the HTF rejection block is too big
Example: **5m RB = ~35 pts** → too large to enter the **wick start** with stop at the full extreme.
| Wick / stop size | Default tactic |
|------------------|----------------|
| **Too big** (e.g. ~35 pts) | Prefer **CE** of the HTF wick/RB; and/or scale to **1m triggers** |
| **Acceptable** (~15–20 pts) with reverse structure inside | Can work off structure / triggers inside the wick |
| Still huge after CE | Pass, micros, or wait for a cleaner LTF invalidation |

Also respect **10 AM** (and other key opens) sitting inside/near the same HTF RB.

### LTF entry-trigger model (inside 5m / 15m RB)
Scale from the HTF block into **1m** and mark what’s **inside**:

| 1m trigger | Role |
|------------|------|
| **IFVG** | Inversion / displacement footprint |
| **CISD** | Change in state of delivery — e.g. bullish = **bullish candle close above a bearish candle** |
| **1m rejection block** | Same RB logic on LTF (already-mitigated 1m RB can still support the 5m story) |

**Signature of a strong HTF RB:** on 1m, **one large displacement candle** that prints **three triggers at once** (1m block + IFVG + CISD). That package, in your bias direction, is a primary way to separate **good vs bad** 5m/15m RBs.

### How to play it
1. Identify HTF RB (5m/15m), note key open confluence if present.  
2. If stop at full wick is too big → CE and/or open 1m.  
3. Map 1m IFVG + CISD + 1m RB inside the HTF zone.  
4. Entries can be:  
   - Limit at HTF **CE** (first tap)  
   - Second tap using overlapping 1m confluences  
   - Pure **1m trigger** with stop at the **tighter 1m swing** (better RR than full 5m wick low/high)  
5. Targets: opposing swing structure, **unfilled balances**, liquidity pools.

### Hard filter example (do not force)
Relative **equal highs** (or similar untaken liquidity) against a 5m RB → **do not take** until that liquidity is **swept**, even if 1m IFVG + CISD already exist inside the block. After a later/cleaner 5m RB forms, re-check the 1m package.

### Precision tradeoff
More precise LTF entries → higher chance of **missing** the move if price never tags the discount/premium trigger. Decide before the session: **max R** or **best fill odds**.

### Allowed entry types / POIs (what you aim at)
Apply one of the **four mechanisms** above to these POIs:
1. **Data H/L opposing-side** after first extreme taken + confirming close + RB  
2. **Soup of key open** after strong close through (18:00 / midnight / 10:00)  
3. **Closest PD array** to the key open  
4. **Rejection block** zone (any TF)  
5. **Wick CE / FVG CE / gap-fill extreme** + engineered liquidity  
6. **10 AM RB / OB** hybrid on 5m, refined on 1m  
7. **CPI/NFP refine:** 5m RB confirm → **1m trigger** out of that 5m RB if the 5m block is huge  
8. **HTF RB → 1m trigger package** (IFVG + CISD + 1m RB / single-candle displacement)  
9. **NWOG / NDOG** tap or fill (prefer close confirm; or enter via fib/OB/RB created at the gap)  
10. **AMD** — limit/trigger the PDA that catches the **manipulation** leg, then ride **distribution**

### Entry choice
| Style | Pros | Cons |
|-------|------|------|
| Limit at HTF wick start | More fills | Wide stop if wick large |
| Limit at HTF CE | Better risk on big RBs | May miss / chop |
| 1m trigger package | Confirmation, tighter stop | Can get edged / left behind |
| Closest 1m PD array | Precise | Can miss tag |

**Default:** if HTF stop is oversized, **CE or 1m triggers**; near open or when confirmation matters, prefer **triggers**; if you already have clean RR on a limit, limits are fine.

## 11. Fibonacci (official rules)

### First principles
1. **Bias first** — same as any PD array: know where price is going before drawing.  
2. Choose a **valid leg** (not random swings). Confirm with HTF when possible (daily RB, equal highs/lows as draw, 1H/4H closes, 15m CISD/RBs).  
3. Preferred working TF for **basic structure + fibs: 5-minute**.  
4. Wait for a **reason to place / update** the fib (e.g. new extreme + opposing close in the active leg). Rebuild the fib when price makes a **new high/low** in the trend.

### Discount / premium discipline
- Prefer entries that actually tag the **discount** (bullish) or **premium** (bearish) region of the fib.  
- A PD array **already in discount** (e.g. 5m gap) can be valid even if the exact discount line is missed by ~**1 pt** — don’t be greedy if you also have an LTF reason (1m RB, sweep into discounted PDA).  
- If a 5m limit at discount gets **edged by a point**, scale to **1m**: if there is a low-risk 1m long/short reason with good RR → take it.

### Best fib levels (limits)
| Level | Stance |
|-------|--------|
| **0.50** | Not preferred for limits |
| **0.62** | **Best-tier** limit level |
| **0.705** | **Best-tier** limit level |
| **0.79** | Deepest he’ll routinely allow; beyond this, **resweep risk rises** (for bullish fib = too deep discount / failure risk) |

**Limit construction examples (bullish fib):**
- Limit **0.62**, stop below **0.705** → ~**1:5**  
- Limit **0.62**, stop below **0.79** → ~**1:3** (**minimum RR** he’ll use)  
- Limit **0.705**, stop below **0.79** → consistent setup, but higher chance of **getting edged** (reversal from 0.62 without tagging 0.705)

Same idea mirrored for shorts into premium.

### Fib + rejection block (standard combo)
Valid bread-and-butter long example:
- HTF bias bullish  
- Fib discount / 0.62 area  
- **5m structure** + **5m rejection block**  
- Stop ~**13–17 pts** acceptable  
- TP toward **first internal high** (examples **1:6, 1:8, ~1:10**)  
- Can enter RB **wick start** for fills, or **CE** for better entry  
- BE when a logical internal high is taken (1m), if using BE management  

### Opportunity philosophy (important)
Low trade frequency is taught — but **take the good setups that print**.  
Do **not** skip a clean **5m RB + 5m structure + fib discount** just because a “more discounted” fill might exist lower. A small loss on a **1:8–1:10** plan followed by another A setup minutes later is fine. High RR means you can afford the scratch/loss.

### Personal preference fork
| Prefer | Accept |
|--------|--------|
| More fills / more stops | Limit shallower fib levels (0.62) |
| Most discounted entry | Wait 0.705/deeper — risk **never getting filled** (OK if it doesn’t tilt you) |

### Keep it simple
Fibs are versatile when you’re unsure which PDA matters: **slap the fib, check the risk, combine** with key opens, RBs, FVGs, IFVGs, OBs, breakers. Simple is not wrong.

## 12. Gap fill / FVG sensitive zones

### Do not enter “anywhere” in an FVG
Large FVGs (example: **~90 pts** on 4H) are not a free zone. Trade the **sensitive parts** of the PD array.

### Two primary FVG levels
| Level | Role |
|-------|------|
| **CE / 50%** of the FVG | What he **mostly** uses for levels |
| **Gap-fill extreme** | **Bullish FVG → very low** (most discount); **bearish FVG → very high** (most premium). Filling the full inefficiency often sparks the reaction |

Same idea on **wick CE** + engineered liquidity below/above (examples: shorts off liquidity under **4H wick CE**).

### How it behaves
- HTF gap fill can look like a **run-through** on that TF; drop TF and you often still get a real reaction (example **~50 pts**).  
- Works on **any timeframe**; lower TF = finer / better entry.  
- Using **only** the extreme (very low/high) = best entries but **more missed fills**. CE / other PDA confluence = more participation. Personal preference: best price vs more trades.

### Confluence rule (most important)
**Combine gap-fill / FVG levels with liquidity — engineered liquidity preferred, but any clear liquidity stack helps.** Liquidity + PDA is framed as a major win-rate booster. Also stack with key opens, RBs, fibs, etc.

### Wick CE / Wick Theory (Models 5 + 3 + trade breakdowns)
Hunt wick theory on **1H / 30m / 15m** (and **M15 news wicks** for data model 3).

**A+ wick CE short example (mirrored for longs):**
1. Quality HTF/mid-TF wick that **swept liquidity**  
2. Mark wick **CE** and/or wick **OTE**  
3. **Engineered liquidity against the CE** (EQH/EQL style) — **mandatory for CE entries**  
4. Stack complements (daily/weekly OB, true open alignment, data extremes, etc.)  
5. Entry at/near CE (spread buffer OK)  
6. Stop tight (often **~5–10 pts**; sniper examples tighter)  
7. Target key level / opposing data / true open / ~50–60 pts per mechanical card

## 13. Stops, targets, management

### Stops
| Size | Use |
|------|-----|
| **~2–5 pts** | Older sniper style / rare; he now prefers **≥5** as default minimum |
| **~5 pts** | **Current preferred minimum** (still extreme R); key opens / wick CEs / CISD / RB with context |
| **~5–10 pts** | Straight-at-level entries when engineered liquidity is present |
| **~10 pts** | Breaker + eng. liq card default |
| **~10–15 pts** | Mechanical checklist default when limiting LTF entries |
| **&lt; 20 pts** | Acceptable for 10 AM RB hybrids |
| **≥ ~25 pts** | Too big → **CE**, refine on 1m triggers, micros, or skip |
| **~35 pts** (lecture example) | Do **not** enter wick start with full-wick stop — CE and/or 1m package |

Placement ideas from lectures:
- Beyond RB wick / beyond CE of the entry FVG  
- On LTF triggers: stop at the **tighter 1m swing**, not the full 5m wick extreme  
- On wick-CE snipers: tiny stop beyond the micro invalidation; entry may be a tick off CE for spread  
- Breaker model card: **~10 pt** SL  
- Trail: e.g. start ~10 → reaction → ~5 → trail further / BE  

### Targets
- **Minimum RR (general): 1:3**, preferably **1:4+**  
- Mechanical card aim: **~50–60 pts** with avg **~3–4R** expectation  
- Standard hybrid range: **~1:5–1:7** (bread and butter)  
- Fib internal-structure targets often **1:6–1:10** when stop is tight  
- **Minimum RR when limiting fibs: ~1:3** (e.g. 0.62 entry, stop beyond 0.79)  
- Wick-CE snipers can target key opens / true open for **very large R** (example **~25R** with 2pt stop)  
- Key-open soups often aim much larger when expansion is on (examples **1:4**, **1:17**, structural highs/lows, through opposing open)  
- Logical objectives: first internal high/low, true open / key opens, NWOG/NDOG fills, 1m/5m/15m gaps, engineered liquidity, unfinished draws / breaker targets  

### Expectancy
- **5pt stops + big TPs = lower win rate.** Expected.  
- Edge = **consistent R expectancy**, not high hit-rate.  
- Not 100% — some “good-looking” soups/fib RBs are losers; take the loss and continue.  
- Near-misses (edged by a point off discount) → check **1m trigger** before quitting the idea.  
- Low frequency ≠ skip A+ fib/RB presentations; high RR absorbs occasional scratches.  
- Many taught levels ≠ license to overtrade.

### Management
- Trail on reaction when using the 10→5 style  
- BE when Apex-style trailing drawdown demands it  
- Set-and-forget allowed as personal preference (especially vs staring at 1m)

### Daily risk cadence (opening-gaps lecture)
Stick to this day structure:
1. **First trade wins** → **get off** (protect the day)  
2. **First trade loses** → next risk is **50%** (if that’s the last allowed shot, get off)  
3. Keep rules + levels + RR discipline — treat it like a constrained game, not unlimited clicks

## 14. Trade grading

### A+
- Clear **daily→LTF** bias + unfinished draw agrees with direction  
- PDH/PDL or equal H/L story coherent; daily SMT checked when relevant  
- Open proximity / first-manipulate read agrees with entry  
- Strong close through key open **and/or** RB at a clear **POI** after a liquidity sweep  
- **Or** clean data H/L: first side swept + flip confirm + RB + opposing target; FF not one-sided trend-day against you  
- HTF RB with **1m displacement package** (IFVG + CISD + 1m RB in one candle) in bias direction  
- **Fib:** valid leg + discount/premium + 5m structure/RB at **0.62 / 0.705** (or PDA in discount within ~1 pt)  
- **FVG CE or gap-fill extreme** paired with **engineered liquidity**  
- **Wick CE sniper:** HTF wick swept liq + engineered liq against CE + HTF complements + true-open alignment  
- **Unfilled empty NWOG/NDOG** as free bias / fill objective, ideally with other PDA confluence  
- **AMD** into PDA on manipulation leg with clear left-side bias (not fighting HTF premium/discount blindly)  
- Entry at wick start, CE, or LTF trigger with stop not “hateful”; fib stops ideally not beyond **0.79** without accepting failure risk  
- Golden pocket / CISD / engineered liquidity when available  
- For 10 AM hybrid days: good 4H wick + sweep + RB  

### B
- Context OK but grey wick / mild depth toward 0.79 / competing gaps  
- Soft data swing (not prominent) but rules otherwise followed  
- Two-scenario HTF fork (e.g. CE not tapped) still unresolved but entry is with residual draw  
- Valid CE / second-tap play on choppy tape  
- Edged exact discount by ~1 pt but LTF trigger still clean  
- FVG extreme-only plan that misses fill (process OK, participation choice)  
- Mid-quality NWOG retrace tap (not the clean jump-away free bias)  
- **AMD short in daily discount / long in daily premium** (works possible, not A+)  
- Stop acceptable but not tiny; OB vs RB unclear so risk is a compromise  

### C / Skip
- No daily/4H bias work before LTF entries  
- Soup/RB with **no context** while opposing draw is obvious  
- Draw already satisfied and bias no longer supports the trade  
- Ignoring open proximity (forcing far-side sweep first when price is already pressed to the near side)  
- Entering **random mid-FVG** with no CE/extreme/liquidity story  
- Fighting a clearly **empty unfilled NWOG/NDOG** with an opposing “bias” and no exceptional reason  
- Overtrading because “too many levels” are visible  
- Breaking day cadence (pressing size after a win / full size after a loss when rules say 50%)  
- Longing a rebalance **against** clear HTF bearish AMD (or mirror) just because the gap looks “fat”  
- Full-wick stop oversized with no CE / LTF refinement  
- “Easy looking” chase with no scare / no premium-discount edge  
- Blind line fade after level already ran  
- Fading **CPI/NFP** after insanely one-sided beat while price is trending  
- Trading **against** data high/low / ignoring SMT that protects the target side  
- 5m RB into **relative equal highs/lows** **without** sweeping that liquidity first (even if 1m IFVG+CISD exist)  
- Limiting **0.50** as primary plan / forcing past **0.79** without accepting resweep risk  
- Skipping clear A fib+RB setups only to “wait for deeper” and then tilting when missed  
- Pressing against a clear **market maker buy/sell model** without SMT/structure permission  
- Trades you can instantly list 2–3 “reasons it failed” before/as you take them (likely not A)  
- Planned R below **1:3**
